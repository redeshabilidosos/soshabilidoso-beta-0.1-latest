DOCUMENTOS DISPONIBLES
================================================================================

1. DOCUMENTACION_APLICACION.txt
   - Descripción: Documentación técnica completa de la aplicación
   - Contenido:
     * Descripción general del proyecto
     * Arquitectura del proyecto
     * Rutas del frontend (Next.js)
     * Endpoints del backend (Django REST API)
     * Modelos de datos
     * Autenticación y autorización
     * Características principales
     * Configuración y variables de entorno
     * Instrucciones de instalación y ejecución
     * Solución de problemas
     * Notas importantes y contexto
   - Audiencia: Desarrolladores, Arquitectos
   - Tamaño: ~50 KB
   - Tiempo de lectura: 30-45 minutos

2. MANUAL_USO_APLICACION.txt
   - Descripción: Manual práctico de uso de la aplicación
   - Contenido:
     * Guía rápida de inicio
     * Registro e inicio de sesión
     * Navegación de la aplicación
     * Crear y compartir contenido
     * Interactuar con otros usuarios
     * Comunidades
     * Mensajería
     * Búsqueda y descubrimiento
     * Configuración de perfil
     * Guía para desarrolladores
     * Comandos útiles
     * Preguntas frecuentes
     * Atajos de teclado
   - Audiencia: Usuarios finales, Desarrolladores
   - Tamaño: ~40 KB
   - Tiempo de lectura: 25-35 minutos

3. REFERENCIA_TECNICA_RAPIDA.txt
   - Descripción: Referencia técnica rápida para consultas
   - Contenido:
     * Puertos y URLs
     * Estructura de carpetas clave
     * Endpoints principales
     * Modelos principales
     * Autenticación
     * Variables de entorno
     * Comandos importantes
     * Dependencias principales
     * Flujos principales
     * Códigos de estado HTTP
     * Manejo de errores
     * Debugging
     * Optimización
     * Seguridad
     * Testing
     * Deployment
     * Recursos útiles
   - Audiencia: Desarrolladores
   - Tamaño: ~30 KB
   - Tiempo de lectura: 15-20 minutos

4. INDICE_DOCUMENTACION.txt
   - Descripción: Este archivo - Índice de documentación
   - Contenido:
     * Lista de documentos
     * Descripción de cada documento
     * Cómo usar la documentación
     * Mapa de navegación
     * Glosario de términos
   - Audiencia: Todos
   - Tamaño: ~20 KB
   - Tiempo de lectura: 10-15 minutos

GLOSARIO DE TÉRMINOS
================================================================================

API: Application Programming Interface (Interfaz de Programación)
JWT: JSON Web Tokens (Tokens de autenticación)
REST: Representational State Transfer (Arquitectura de API)
CORS: Cross-Origin Resource Sharing (Compartir recursos entre orígenes)
CRUD: Create, Read, Update, Delete (Operaciones básicas)
ORM: Object-Relational Mapping (Mapeo de objetos a BD)
UUID: Universally Unique Identifier (Identificador único)
FK: Foreign Key (Clave foránea)
BD: Base de datos
HTTP: HyperText Transfer Protocol
HTTPS: HTTP Secure
WebSocket: Protocolo de comunicación bidireccional
JSON: JavaScript Object Notation
XML: eXtensible Markup Language
SQL: Structured Query Language
NoSQL: Not Only SQL
MVC: Model-View-Controller
MVP: Model-View-Presenter
MVVM: Model-View-ViewModel
SPA: Single Page Application
SSR: Server-Side Rendering
CSR: Client-Side Rendering
SEO: Search Engine Optimization
PWA: Progressive Web Application
CI/CD: Continuous Integration/Continuous Deployment
DevOps: Development Operations
Git: Sistema de control de versiones
npm: Node Package Manager
pip: Python Package Installer
venv: Virtual Environment (Python)
Docker: Contenedorización
Kubernetes: Orquestación de contenedores
